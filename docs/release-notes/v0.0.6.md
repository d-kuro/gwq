# Release v0.0.6

## ‚ú® New Features

### Repository Settings for Per-Repository Setup (#41)

- Added `repository_settings` configuration to customize worktree creation per repository
- **File Copying**: Automatically copy files from repo root to new worktrees using `copy_files` option (supports glob patterns)
- **Setup Commands**: Run setup commands in new worktree directory using `setup_commands` option
- Errors are logged but do not abort worktree creation

Example configuration:

```toml
[[repository_settings]]
repository = "~/src/myproject"
copy_files = ["templates/.env.example"]
setup_commands = ["npm install"]
```

### Remove Task Command (#45)

The `gwq task` command was introduced as an experimental feature for automated development task management with Claude Code. After evaluation, we decided to remove this feature to keep gwq focused on its core functionality as a git worktree manager.

- Removed `gwq task` command and all related code
- Deleted `internal/claude/` package (task management, execution engine, etc.)
- Deleted `internal/tui/` package (log viewer)
- Removed `bubbletea` dependency
- **Migration**: Users relying on `gwq task` should use alternative approaches such as shell scripts or other task management tools
