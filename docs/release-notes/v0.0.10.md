### Local Config File Support (#59)

Added support for project-specific configuration via `.gwq.toml` files. Local config takes precedence over global settings (`~/.config/gwq/config.toml`).

**Usage:**

```bash
# Set local configuration (writes to .gwq.toml in current directory)
gwq config set --local finder.preview false

# View merged configuration
gwq config list
```

**Features:**

- `.gwq.toml` in the current directory is automatically loaded and merged
- Local settings override global settings
- `repository_settings` are merged by `repository` field as key
- New `--local` flag for `gwq config set` command

### repository_settings Merge Behavior

When both global and local configs define `repository_settings`:

- **Same repository**: Local settings completely override global
- **Different repositories**: Both are kept
