### `cd.launch_shell` Option (#80)

Added a `cd.launch_shell` config option (default: `true`) that allows `gwq cd` to change directory in the current shell without spawning a nested shell.

**Configuration:**

```toml
[cd]
launch_shell = false  # default: true
```

**Setup:**

```bash
# bash (~/.bashrc)
source <(gwq completion bash)

# zsh (~/.zshrc)
source <(gwq completion zsh)

# fish (~/.config/fish/config.fish)
gwq completion fish | source
```

When `launch_shell = false`, the shell wrapper function intercepts `gwq cd` and runs `cd` in the current shell instead of spawning a nested shell. Supports bash, zsh, and fish. PowerShell is completion only.
